# wq-gzh-marketing：公众号营销植入

营销植入不是结尾加一句"欢迎咨询"。植入的位置、强度、方式，决定它是内容的一部分还是破坏阅读体验的广告。

---

## 第一步：确认三个参数

```yaml
转化意图: 无 / 软植入 / 强转化
目标动作: 无 / 评论 / 私信 / 加微信 / 领取资料 / 报名 / 咨询 / 购买
账号: 破局邦 / 惟乔内容获客 / 写作有光
```

账号默认转化意图：

| 账号 | 默认 |
|---|---|
| 破局邦 | 无，必要时软植入 |
| 惟乔内容获客 | 软植入，产品期可强转化 |
| 写作有光 | 无或软植入 |

---

## 四类组件

### 组件1：陪跑案例软植入（最重要）

**核心规则：案例即论证。**
案例必须严丝合缝证明本文核心论点，不是为了展示服务插进去的。

植入格式：
```
背景一句 → 做了什么改变一句 → 具体结果数字 → 配带水印截图
```

硬规则：
- 单个案例 ≤100字，嵌入论证段落，不独立成 section
- 紧跟论点之后作为例证，不放开头
- 一篇最多 2 个陪跑案例
- 正文不写"我有陪跑服务"等推销句，广告信息交给截图水印
- 没有真实数字、具体动作、截图路径的案例，不植入

案例归档位置：`01-素材/案例库/陪跑案例/`

写法示例：
```
我有个做教培的学员，之前发课程通知没人看。
改成发"家长最怕踩的3个坑"之后，第一篇阅读2000+，暑假班当天来了2个家长报名。
（配带水印截图）
```

### 组件2：痛点共鸣

作用：让读者确认"这说的是我"。

| 强度 | 写法 | 位置 |
|---|---|---|
| 无 | 描述现象，不指向解决方案 | 开头、观点段 |
| 软植入 | 点名解决方向，不展开服务 | 开头、中段 |
| 强转化 | 明确解决方案存在，预告获取方式 | 开头、转化段 |

### 组件3：信任背书

作用：回答"凭什么信你"。

| 强度 | 写法 | 位置 |
|---|---|---|
| 无 | 纯内容价值，不出现个人背书 | 方法段、观点段 |
| 软植入 | 个人经历、理念一致性 | 中段、结尾 |
| 强转化 | 资质、战绩、服务边界、筛选标准显性化 | 转化段、文末 |

### 组件4：行动诱导

作用：让读者知道下一步做什么。

| 强度 | 写法 | 位置 |
|---|---|---|
| 无 | 开放式收束，不出现商业动作 | 文末 |
| 软植入 | 轻互动，留反馈或私信入口 | 文末 |
| 强转化 | 明确下一步，三限原则：限时、限量、限定人群 | 文末或文中 |

强转化模板：
```
如果你是__，现在卡在__，可以添加微信/私信__，发送"__"领取/报名/咨询。仅限__，截止到__。
```

## 输出格式

配置组件时输出：

```markdown
转化意图：软植入
目标动作：私信/加微信
组件配置：
- 案例锚定：软植入，放在第二节论点证明处（陪跑案例：行业-关键词.md）
- 痛点共鸣：软植入，放在开头第二段
- 信任背书：无
- 行动诱导：软植入，文末

植入草稿：
[各组件的具体文字]

风险提醒：[是否太硬/太软/跑偏/截图缺失]
```

---

## 发布前检查

- 转化意图为"无"时，完全没有产品、服务、咨询入口
- 软植入自然，不像广告
- 陪跑案例严格证明当前论点，≤100字，嵌入段落，一篇不超过2个
- 陪跑案例有真实数字、具体动作和截图路径
- 强转化明确人群、痛点、结果、行动方式
- 未编造价格、名额、截止时间、成果数据
