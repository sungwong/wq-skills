# wq-gzh-analytics：公众号数据分析

用户把公众号后台截图发过来，AI直接读图，不需要手动填数字。

---

## 使用方式

1. 文章发布后24-48小时，打开公众号后台
2. 截取文章数据页截图（包含阅读量、完读率、分享、在看等）
3. 把截图发给AI，说「分析一下」

---

## 分析框架（基础版）

收到截图后，读取以下核心指标并给出判断：

**流量指标**
- 阅读量：绝对值 + 与账号均值对比
- 来源分布：推荐流 vs 粉丝 vs 分享（哪个渠道来的人最多）

**质量指标**
- 完读率：判断内容是否留住了人
- 在看/分享：判断内容是否值得传播
- 涨粉数：判断内容是否吸引了新读者

**综合判断**
输出一个结论：这篇文章属于哪种表现？
- 爆款（阅读远超均值，完读率高）
- 正常（符合账号均值）
- 低于预期（阅读低或完读率低）
- 流量好但质量差（阅读高但完读低，说明标题骗了人）

---

## 输出格式

```
【数据读取】
- 阅读量：
- 完读率：
- 分享/在看：
- 涨粉：

【核心判断】
这篇属于：[爆款/正常/低于预期/流量好质量差]

【原因推测】
[表现好/差的可能原因，结合选题、标题、开头判断]

【下一步建议】
[是否值得复用这个选题/结构/标题风格]
```

## 细化分析（暂缓）

以下维度先不做，后续有需要再补：
- 分时阅读曲线
- 用户画像分析
- 跨文章横向对比
- 关键词/话题表现追踪
