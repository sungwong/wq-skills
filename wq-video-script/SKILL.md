# wq-video-script：短视频脚本写作

短视频和公众号文章有一个根本差别：公众号读者已经点进来了，短视频用户随时会划走。所以短视频的每一秒都在争留存，不只是开头。

---

## 使用方式

用户给一个主题或想法，直接出完整脚本，不分步确认。缺少参数时按默认值处理，不追问。

**信息不足时先对话：** 如果用户只给了一个方向，不清楚写给谁或营销目的，先按对话引导原则问清楚再动笔。参考：`02-方法论/写作系统/对话引导原则.md`

默认值：
- 平台：抖音
- 时长：1-3分钟
- 账号：惟乔内容获客
- 转化意图：软植入
- 行动钩子：收藏型

---

## 五个结构要素

### 要素1：钩子（前3秒）

短视频钩子不只是开头一句话，它贯穿全片——开头钩住人、中间钩住人继续看、结尾钩住人做动作。

**开头钩子（前3秒）：**

两个条件必须同时满足：
- **场景关联**：让目标用户立刻感到"这说的是我"
- **反常识**：让人产生"等等，这和我想的不一样"

禁止开头：
- 抽象无关的大道理（"在这个时代……"）
- 自我介绍（"大家好，我是……"）
- 铺垫背景（"今天我想聊一个话题……"）

开头钩子模板：
```
[目标用户场景] + [反常识判断]
例："很多人以为公众号没流量是因为不会写，其实是因为写错了方向。"
```

**中间钩子（承转处）：**
- 每隔30-45秒埋一个"还有更重要的"或"但真正的问题是……"
- 悬念不用说破，让观众猜

**结尾钩子（行动）：**
- 明确一个动作：点赞/评论/关注/私信
- 给一个理由，不只是"记得关注我"

---

### 要素2：骨架

**知识类短视频必须做显性化骨架。**

知识类内容（方法论、干货、认知类）要在视频开头或片头字幕卡里把骨架亮出来，让观众一眼看到"你要讲几点、讲什么"。

三个作用：
- **显性化框架**：一眼看懂结构，不用猜
- **字幕条提示**：每进入一个新段落，屏幕上出现"第一点/第二点"提示，观众知道自己在哪、还剩多少
- **降低认知负荷**：观众专注理解内容，不用分心追"这人说到哪了"

禁止做"意识流"：不能直接就开讲，让观众自己猜结构。意识流适合情绪类、故事类，不适合知识类。

骨架示例（字幕卡/口播）：
```
"今天讲3件事：流量机制、用户心态、变现逻辑。先讲第一个——"
```

---

短视频骨架比公众号更短更紧，常用三种：

**A. 问题-根因-解法**
```
开头抛问题 → 揭示真正原因（反常识）→ 给出解法 → 行动
```
适合：方法论、干货、认知类

**B. 反常识-论证-收口**
```
先立一个大多数人不信的判断 → 用案例/数据证明 → 收一个让人记住的金句
```
适合：观点类、认知颠覆类

**C. 故事-转折-提炼**
```
一个具体的人遇到具体的事 → 意外转折 → 提炼成普遍规律
```
适合：案例类、情绪共鸣类

---

### 要素3：情绪弧线

短视频情绪不能平，要有起伏：

- **开头**：制造张力（好奇/共鸣/不适）
- **中间**：加深/转折（认知被挑战）
- **结尾**：释放（豁然开朗/认同/被点醒）

**情绪弧线的标准曲线（待验证）：**

- **前1/3：认知高潮** — 情绪强度最高点要提前，不能放在中间或结尾。开头钩子之后立刻给出最反常识、最有冲击力的判断，让观众在前1/3就被震到。这是完播率的关键，观众决定"要不要看完"就在这个峰值附近。
- **中段：缓坡下行** — 情绪强度自然下降，但不能掉太快。用论证、案例、细节托住，让观众处于"理解消化"状态而不是"等结束"状态。
- **结尾：释然感** — 不需要再次拉高情绪，而是给一种"终于想通了/原来如此"的释放感。认知收束做好了，释然感自然有。

这条曲线的数据表现：完播率高、完播深度深。（此为参考框架，需跑实际数据验证。）

情绪弧线靠两件事驱动：
1. **故事和案例**——具体的人、具体的数字、具体的场景（找`/wq-gzh-story`）
2. **节奏**——长句给信息，短句给冲击；密集处紧张，停顿处喘气

---

### 要素4：认知收束

视频结束前必须有一句让人记住的判断，不是总结，是**一句能被传播的话**。

要求：
- 有明确立场，不模糊
- 能脱离视频单独存在
- 最好有点反常识（找`/wq-gzh-viewpoint`打磨）

---

### 要素5：行动闭环

互动不是求来的，是给用户一个无法拒绝的参与理由。三种钩子选一个，不要叠加：

**收藏型钩子**（适合干货/知识类）
给用户一个收藏的具体理由，不只是"怕找不到"：
```
我把本期涉及的参考文献和数据来源做成了一个知识卡片，对照着看效果更好，建议先收藏以免划走找不到。
```

**讨论型钩子**（适合观点/共鸣类）
抛一个具体问题让用户有话可说，不是泛泛的"你怎么看"：
```
在你的工作场景里有没有遇到过案例中的情况？可以把经历在评论区写出来，我们一起拆解。
```

**复访型钩子**（适合系列内容）
预告下期并制造期待，让用户觉得不关注会亏：
```
这只是入门篇，下期我们用这个模型来拆解最近的行业热点，它的逻辑简直一模一样。点个关注别错过。
```

---

## 第三步：脚本格式

输出标准脚本格式，分镜头说明可选：

```markdown
【标题/封面文字】（≤10字，即使没声音也能吸引点击）

【开头钩子】（前3秒，口播文字）

【主体】
[段落1]
[段落2]
[段落3]

【结尾金句】（认知收束，一句话）

【行动引导】（一个动作）

---
时长预估：XX秒
节奏提示：[哪里停顿/哪里加速/哪里降调]
```

## 与公众号的核心差异

| | 短视频 | 公众号 |
|---|---|---|
| 钩子压力 | 极高，前3秒生死 | 较低，已点进来 |
| 篇幅 | 150-500字口播 | 800-2000字 |
| 情绪节奏 | 密集，每30秒一个节点 | 可以慢一些 |
| 开头方式 | 场景+反常识 | 论理式直写（默认） |
| 故事用法 | 可以是主线 | 只是论证节点 |
| 认知密度 | 一个核心观点就够 | 可以有3-5个论点 |
